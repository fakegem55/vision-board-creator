# Vision Board Creator — Requirements

> 서비스 한 줄 요약: 키워드 몇 개와 한 줄 다짐만 입력하면, 나만의 비전보드가 자동으로 완성되는 모바일 웹 서비스

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 대상 사용자 | 디지털 기기가 낯선 사용자, 비전보드를 처음 만들어보는 사람 |
| 제공 채널 | 모바일 웹 (반응형) |
| 핵심 가치 | **단순함** — 입력 2개, 버튼 1개로 완성 |
| 저장/공유 | 기기 로컬 저장 (PNG 다운로드) + OS 공유 시트 (카카오톡 등) |

---

## 2. 사용자 흐름 (Happy Path)

```
① 키워드 입력 (1~10개 태그)
      예: "여행" "건강" "가족" "성공"
② 한 줄 다짐 텍스트 입력
      예: "2025년, 나는 매일 성장한다"
③ "내 비전보드 만들기" 버튼 탭
      → 키워드별 이미지 자동 수집 (Unsplash)
      → 이미지 + 텍스트 자동 배치
④ 완성된 보드 확인
      → PNG 저장 (카메라롤)
      → 공유 (카카오톡·문자 등 OS 공유 시트)
      → 재생성 (같은 키워드, 다른 이미지 조합)
```

---

## 3. User Stories

### US-01 키워드 입력
> **As a** 비전보드를 만들고 싶은 사용자,
> **I want to** 내 목표와 관련된 키워드를 태그 형태로 입력하고 싶다,
> **So that** 무엇을 디자인할지 고민하지 않고 바로 시작할 수 있다.

**Acceptance Criteria:**
- [ ] 텍스트 입력 후 Enter 또는 스페이스로 키워드 태그가 생성된다
- [ ] 태그는 최소 1개, 최대 10개까지 입력 가능하다
- [ ] 태그를 탭하면 삭제할 수 있다
- [ ] 10개 초과 시 입력이 차단되고 안내 메시지가 표시된다

---

### US-02 다짐 텍스트 입력
> **As a** 나의 다짐을 보드에 담고 싶은 사용자,
> **I want to** 한 줄 문장을 입력하고 싶다,
> **So that** 이미지와 함께 나의 의지를 시각적으로 표현할 수 있다.

**Acceptance Criteria:**
- [ ] 텍스트 입력 필드 1개가 제공된다
- [ ] 최대 50자 제한이 있다
- [ ] 입력하지 않아도 보드 생성이 가능하다 (선택 사항)

---

### US-03 비전보드 자동 생성
> **As a** 디자인을 모르는 사용자,
> **I want to** 버튼 하나로 보드가 자동으로 만들어지길 원한다,
> **So that** 레이아웃이나 디자인을 직접 고민하지 않아도 된다.

**Acceptance Criteria:**
- [ ] "만들기" 버튼 탭 시 키워드별로 Unsplash 이미지를 1장씩 가져온다
- [ ] 이미지들이 보드에 자동으로 배치된다 (그리드 또는 콜라주 레이아웃)
- [ ] 다짐 텍스트가 보드 내 적절한 위치에 표시된다
- [ ] 로딩 중 스피너 또는 스켈레톤이 표시된다
- [ ] 이미지 로드 실패 시 대체 이미지(placeholder)가 표시된다

---

### US-04 보드 재생성
> **As a** 결과가 마음에 들지 않는 사용자,
> **I want to** 같은 키워드로 다른 이미지 조합을 시도하고 싶다,
> **So that** 내 취향에 맞는 결과를 찾을 수 있다.

**Acceptance Criteria:**
- [ ] "다시 만들기" 버튼 탭 시 같은 키워드로 새 이미지 조합이 생성된다
- [ ] 이전과 다른 이미지가 선택된다 (랜덤 오프셋 적용)

---

### US-05 PNG 저장
> **As a** 완성한 비전보드를 기기에 저장하고 싶은 사용자,
> **I want to** 보드를 이미지 파일로 다운로드하고 싶다,
> **So that** 배경화면이나 앨범에서 언제든 볼 수 있다.

**Acceptance Criteria:**
- [ ] "저장" 버튼 탭 시 보드가 PNG 파일로 다운로드된다
- [ ] 파일명은 `vision-board.png` 형식이다
- [ ] 모바일 Safari/Chrome에서 카메라롤 저장이 동작한다

---

### US-06 공유
> **As a** 비전보드를 주변에 보여주고 싶은 사용자,
> **I want to** OS 공유 시트를 통해 이미지를 공유하고 싶다,
> **So that** 카카오톡·문자 등으로 바로 전달할 수 있다.

**Acceptance Criteria:**
- [ ] "공유" 버튼 탭 시 Web Share API (navigator.share)가 호출된다
- [ ] 미지원 브라우저에서는 PNG 다운로드로 폴백된다

---

## 4. 비기능 요구사항

| 구분 | 요구사항 |
|------|---------|
| 성능 | 이미지 로드 포함 보드 생성 5초 이내 |
| 접근성 | 터치 타겟 최소 44×44px |
| 반응형 | 모바일 우선 (360px~) |
| 브라우저 | iOS Safari 16+, Android Chrome 110+ |

- 드래그&드롭 수동 편집
- 스티커, 색상 커스터마이징
